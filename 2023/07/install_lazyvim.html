    安装LazyVim

    1. 前言

    主要记录我在安装 LazyVim 时踩的一些坑

    2. 最好不要使用mingw-w64

    LazyVim 中的一个插件 nvim-treesitter 需要 C compiler, 如果在网上搜索怎么配置 C compiler 的环境, 我相信出现的最多的结果就是安装 mingw-w64. 但就我自己的经验为依据, 网上大部分的教程是过时的, mingw-w64 的官网的下载页面应该是改变过布局, 如果是小白, 可能很难找到正确的安装包, 比如我最后就下载了一个 MinGW, 其中有一个 mingw-get.exe, 我在这个exe中下载了 x86-32 版本的 gcc 及其相关依赖, 然后 nvim-treesitter 就一直显示 compile failed, Google了几个小时才解决问题.

    参考:
    This was the culprit Mingw doesn’t work! get scoop!
    Failed to load parser, parser.so is not a valid Win32 app · Issue 438 · nvim-treesitternvim-treesitter
    Error executing lua Failed to load parser uv_dlopen cpp.so is not a valid Win32 application. · Issue 1985 · nvim-treesitternvim-treesitter

    推荐使用 scoop 安装 gcc 和 g++, scoop 中安装软件非常简单

    3. 如果一直遇到报错, 可以尝试删除nvim-data文件夹

    如果在使用 LazyVim 时一直报错, 可能是因为在下载插件时因为网络问题或者依赖不完整导致 nvim-data 中的数据不完整, 而 LazyVim 不能自动修复, 因此需要我们在确保依赖完整的情况下, 删除 nvim-data 文件夹, 重新下载插件

    windows中 nvim-data 一般位于 C:\Users\jiang\AppData\Local\nvim-data

    官网推荐的依赖:

    • Neovim >= 0.8.0 (needs to be built with LuaJIT)
    • Git >= 2.19.0 (for partial clones support)
    • a Nerd Font(v3.0 or greater) (optional, but needed to display some icons)
    • lazygit (optional)
    • a C compiler for nvim-treesitter. See here
    • for telescope.nvim (optional)
      • live grep: ripgrep
      • find files: fd
    • a terminal that support true color and undercurl:
      • kitty (Linux & Macos)
      • wezterm (Linux, Macos & Windows)
      • alacritty (Linux, Macos & Windows)
      • iterm2 (Macos)

    3.1. 报错示例

    gcc.exe: fatal error: cannot execute 'cc1': CreateProcess: No such file or directory

    删除 nvim-data 文件夹, 确保C语言环境正确后, 然后重新打开
