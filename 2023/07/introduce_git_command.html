    1. 前言

    这是一份个人使用 git 时的 CheatSheet, 如果您希望更浏览详细的教程, 请参考以下的教程:
    Git 教程 菜鸟教程

    2. 设置全局配置

    git config --global user.name "JiangYun"
git config --global user.email "jiangyunacw@gmail.com"

git config --global core.quotepath false 
# 如果此项为"true", 在使用"git status"命令时, 中文会被转义, 看起来像乱码.
# 参考: https://segmentfault.com/a/1190000020807726

git config --global init.defaultbranch main       # 设置默认初始分支为"main", 与Github相匹配
git config --global core.autocrlf false               # 禁用Git在检出和提交文件时对换行符的自动转换. 可消除烦人的 warnings.

    上面这一步一般在刚安装git后进行

    • 查看全局配置
    git config --global --list --show-origin

# 解释参数:
# --list  查看全局配置
# --show-origin (可选) 展示".gitconfig"的位置

    更多的有关全局配置的教程请参考: How to set important Git config global properties

    • 取消全局配置
    git config --global --unset <variable>

    3. 创建版本库

    git init

    4. 提交更新

    git add --all
git add specific-file
git status
git commit -m "some note"

# 一次性更新
git add --all; git status; git commit -m "batch update"

    5. 切换版本

    # 回退到上一个版本
git reset --hard HEAD^
# HEAD表示当前版本, HEAD^表示上一个, HEAD^^表示上上个

# 根据版本号, 切换版本
git reset --hard <版本号>

    6. 其他命令

    • 查看提交历史
    git log
git log --pretty=oneline # 每个记录只显示一行

    7. 和远程仓库(GitHub)交互

    本地电脑 (Windows) 第一次和GitHub交互前, 检查 C:\Users\your-username\.ssh 目录下有没有 id_rsa 和 id_rsa.pub 文件. 如果没有,

    1. 打开 Git Bash, 运行: ssh-keygen -t rsa -C "jiangyunacw@gmail.com", 一路回车
    2. 登录GitHub后, 打开网页https://github.com/settings/keys, 点击 New SSH Key, 填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容, 保存

    7.1. 克隆远程仓库

    git clone someURL
# 推荐ssh协议, 它速度更快
